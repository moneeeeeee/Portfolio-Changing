Grid Migration Simulator

Simulation Developer | Java, Gradle, JUnit

Overview

I designed and implemented a city simulation in Java to explore how agents migrate through a grid-based environment based on satisfaction, crowding, and fairness. Each agent represents a resident evaluating their surroundings to decide whether to stay, move, or relocate.

Tools

  • Java (object-oriented design)
  • Gradle (build automation)
  • JUnit (testing)

Role

Simulation Developer

Description

The simulation used modular classes for agents, buildings, walkways, and time tracking. I implemented a two-step update system to ensure consistency in agent decision-making across the grid. Agents evaluated satisfaction thresholds, reset conditions, and density limits when deciding whether to move. I integrated Gradle for build automation and JUnit to test movement logic and system stability.

To analyze outcomes, I built custom TimeSeries objects to track agent behavior and placement over time, which revealed emergent patterns such as congestion and clustering. This project showed how local decision-making scales into large system behavior — a concept directly relevant to distributed systems, smart city planning, and load balancing.
